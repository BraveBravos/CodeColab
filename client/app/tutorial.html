Merge Arrow

At any point in your code, if you wish to revert changes you can click the white arrow in the middle of the left and right panel. Multiple arrows will show up, each one representing a different section of changes so you will be able to have complete control over what you wish to revert.

Committing

Once you select a repository, a 'CODECOLAB' branch is automatically created and a commit button will appear in the nav bar. Clicking the commit button, you are commiting your changes in the left panel to the GitHub branch- not to master. We've done this to protect any previous code and to keep the traditional GitHub workflow behind the scenes. You can delete the branch at any point on GitHub if you don't wish to keep the changes you've committed and when opening the repository again in CodeColab, a new branch will be recreated.

Pulling / Merging

Once your team is satisfied with the changes and commits them, a merge button will appear in the right of the nav bar. Clicking this will prompt you to enter a pull request message. Entering this will submit a pull request from the CodeColab branch to master, and then merge the pull request in to the master branch of the repository after a second security prompt to confirm the merge.

Deployment

Once changes have been commited and merged into master, you can deploy your application on Heroku by clicking the 'Deploy' button in the upper right hand corner. You will be prompted to enter the name of your application, and your newly deployed application will open in separate tab.
